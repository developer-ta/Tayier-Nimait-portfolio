# Collecte portfolio - projets prioritaires

_Objectif : extraire uniquement les informations utiles pour construire la page Projets du portfolio._

## 1. Ce qu'on garde en tete

Le portfolio doit montrer :

- une competence visible,
- une solution concrète,
- une utilité marché,
- une capacité à livrer seul,
- une version publique sans divulguer les détails sensibles.

## 2. Projet prioritaire n°1

### Smart Facturation AI

**Nom public conseillé**

> Agent IA de facturation pour PME

**Pourquoi il est prioritaire**

- automatisation métier très lisible,
- utile pour alternance, freelance et entreprise,
- parle à la fois à un recruteur et à un client,
- facile à comprendre vite.

**Ce qu'il faut montrer**

- compréhension d'une demande utilisateur,
- orchestration des tâches de facturation,
- structure backend / frontend,
- logique de contrôle traçable,
- docs de cadrage et de planning.

**Ce qu'il faut cacher**

- la logique interne trop détaillée,
- les documents de travail sensibles,
- les prompts trop précis,
- la mécanique exacte de l'orchestration.

**Mots-clés à garder**

- IA appliquée
- automatisation métier
- facturation
- PME
- workflow
- contrôle
- traçabilité
- backend
- frontend

**Forme de carte portfolio**

- Titre : `Agent IA de facturation pour PME`
- Sous-titre : `Automatisation des tâches métier en langage naturel`
- Impact : `réduction du travail manuel et meilleur contrôle`
- Badge : `MVP`

---

## 3. Projet prioritaire n°2

### Dialogue Digest Agent core

**Nom public conseillé**

> Agent de structuration de dialogues en projet exploitable

**Pourquoi il est prioritaire**

- montre la logique d'agent,
- montre la transformation de dialogue brut en structure,
- très bon pour IA appliquée et pipeline,
- peut servir de preuve d'architecture.

**Ce qu'il faut montrer**

- découpage de dialogue,
- génération de briques structurées,
- préparation de fichiers d'analyse,
- orchestration par étapes,
- logique de sessions.

**Ce qu'il faut cacher**

- les prompts complets,
- les archives internes,
- les données sensibles de dialogue,
- les versions trop expérimentales.

**Mots-clés à garder**

- agent IA
- structuration
- dialogue
- pipeline
- orchestration
- prompt
- analyse
- automatisation

**Forme de carte portfolio**

- Titre : `Agent de structuration de dialogues`
- Sous-titre : `Transformation de dialogues bruts en briques de projet`
- Impact : `gagne du temps dans l'analyse et l'organisation`
- Badge : `Prototype`

## 4. Projet prioritaire n°3

### Solena

**Nom public conseillé**

> Vitrine publique d'un workflow de développement assisté par IA

**Pourquoi il est prioritaire**

- très bon pour montrer la vitrine publique,
- montre le split public / privé,
- montre la capacité à faire un produit clair,
- bon signal pour grande société et freelance.

**Ce qu'il faut montrer**

- landing page publique,
- workflow de développement,
- desktop MVP,
- launcher,
- séparation public / privé,
- structure propre du dépôt.

**Ce qu'il faut cacher**

- la mémoire privée,
- les dialogues bruts,
- les notes internes,
- la couche de gouvernance sensible.

**Mots-clés à garder**

- vitrine publique
- workflow
- desktop MVP
- launcher
- développement assisté par IA
- séparation public / privé

**Forme de carte portfolio**

- Titre : `Vitrine publique d'un workflow IA`
- Sous-titre : `Projet vitrine pour développement assisté par IA`
- Impact : `montre méthode, clarté et capacité de structuration`
- Badge : `Version active`

## 5. Projet prioritaire n°4

### Python-Mastery-Nexus-Triple-Core-main

**Nom public conseillé**

> Écosystème Python fullstack, data et IA industrielle

**Pourquoi il est prioritaire**

- montre un niveau technique large,
- montre Python + data + UI + desktop,
- bon pour un recruteur qui cherche un profil solide,
- sert de preuve de maturité technique.

**Ce qu'il faut montrer**

- architecture globale,
- backend Python / Django,
- data pipeline,
- frontend / templates,
- launcher desktop,
- gouvernance IA.

**Ce qu'il faut cacher**

- le langage trop auto-promotionnel,
- les détails trop lourds pour la page publique,
- les parties expérimentales non stables.

**Mots-clés à garder**

- Python
- Django
- data
- UI
- desktop app
- architecture logicielle
- gouvernance IA

**Forme de carte portfolio**

- Titre : `Écosystème Python fullstack et data`
- Sous-titre : `Architecture industrielle pour web, data et desktop`
- Impact : `montre une capacité à structurer une solution complète`
- Badge : `Architecture validée`
