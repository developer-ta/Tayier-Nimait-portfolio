# GPS Documentaire 2026

_Carte de navigation pour retrouver rapidement les documents du portfolio sans supprimer ni déplacer l'existant._

## 1. Comment lire ce dossier

Tu peux penser ce dossier comme un GPS :

- `INDEX.md` = porte d'entrée stable pour les documents de référence
- `GPS_INDEX.md` = carte globale de navigation
- `source/` = matières premières, notes, brouillons, contenu de base
- `design/` = documents liés au design, à l'UI et aux assets
- `prompts/` = historiques de prompts, prototypes de contenu, images et variantes
- `releases/` = versions datées et validées

## 7. Versioning

Format recommandé :

`YYYY-MM-DD_vX`

Exemples :

- `2026-05-03_v1`
- `2026-05-10_v2`
- `2026-04-21_v3`

Règle simple :

- la date dit quand la version a été produite,
- `vX` dit l'ordre de la version,
- plus le numéro est élevé, plus la version est récente dans la même date ou la même série.

## 9. Ce qu'il ne faut pas faire

- supprimer les anciens fichiers,
- renommer sans garder de trace,
- mélanger brouillon et validé dans la même zone,
- multiplier les copies sans index,
- écrire des documents sans les rattacher à une version.

## 10. Règle d'or

Chaque nouveau document important doit pouvoir répondre à ces 3 questions :

1. À quelle date a-t-il été produit ?
2. Quelle est sa version ?
3. Dans quelle catégorie de travail doit-il être retrouvé ?
