# Planning pour remplir la page Projets du portfolio

_Objectif : transformer des repos bruts en 3 projets présentables, sans se perdre dans tout le code._

## 1. Mon avis sur le document analyse

Oui, l'idée générale est bonne.

Le pipeline proposé est cohérent :

- repo brut,
- extraction,
- transformation,
- format portfolio.

Mais pour avancer vite, il faut le rendre plus simple.

### Ce que je valide

- partir des repos réels,
- extraire les éléments visibles,
- transformer la technique en langage métier,
- afficher seulement les projets utiles.

### Ce que je simplifie

- pas besoin de traiter 100 projets,
- pas besoin de tout documenter,
- pas besoin de montrer le code brut dans le portfolio.

## 2. Objectif concret du jour

Trouver les informations utiles pour remplir la page Projets du portfolio.

Tu dois produire :

1. les 3 projets principaux,
2. les résumés courts,
3. les résultats visibles,
4. les éléments de preuve,
5. l'ordre d'affichage.

## 3. Pipeline simple à suivre

### Étape 1 - Repérer

Tu parcours les repos et tu retiens seulement les projets qui ont :

- une vraie interface,
- une vraie automatisation,
- un usage métier clair,
- une preuve visuelle.

### Étape 2 - Extraire

Pour chaque projet retenu, tu récupères :

- README.md,
- fichiers d'entrée principaux,
- nom du projet,
- stack,
- fonctionnalité visible,
- usage réel.

### Étape 3 - Transformer

Tu transformes le projet technique en texte simple :

- titre court,
- phrase courte,
- impact crédible,
- badge,
- ce qu'on montre visuellement.

### Étape 4 - Classer

Tu gardes seulement :

- Top 3 pour la page d'accueil,
- les autres vont plus bas ou en archive.

## 4. Ordre de travail recommandé

### Bloc A - Sélection

Choisir les projets qui ont le plus de valeur pour le marché.

Critères :

- visibilité,
- utilité métier,
- présence d'IA ou d'automatisation,
- capacité à faire une démo.

### Bloc B - Lecture

Lire pour chaque projet :

- README,
- structure du dossier,
- fichiers d'entrée,
- documents de version.

### Bloc C - Synthèse

Écrire pour chaque projet :

- nom du projet,
- problème,
- solution,
- résultat,
- stack,
- preuve visuelle.

### Bloc D - Mise en page

Transformer la synthèse en carte portfolio :

- titre,
- sous-titre,
- impact,
- badge,
- bouton ou lien.

## 5. Modèle de fiche projet à remplir

Tu peux remplir chaque projet avec ce format :

```text
Nom du projet :
Type :
Problème :
Solution :
Résultat :
Stack :
Preuve visuelle :
Badge :
Priorité :
```

## 6. Score pour choisir les 3 meilleurs

Tu peux noter chaque projet sur 10 :

- 4 points pour la valeur métier,
- 3 points pour la clarté visuelle,
- 3 points pour la présence d'IA / automatisation.

### Règle

- score 8 à 10 : projet principal
- score 6 à 7 : projet secondaire
- score inférieur à 6 : archive ou plus tard

## 7. Format de sortie souhaité

Pour le portfolio, chaque projet doit finir en version courte :

```text
Titre court
Phrase simple
Impact crédible
Badge
Ce qu'on voit
```

### Exemple

- `Automatisation de données`
- `Pipeline qui nettoie et prépare les données`
- `Réduit le travail manuel et fiabilise le traitement`
- `MVP`
- `Dashboard + flux automatisé`

## 8. Ce qu'il faut regarder dans les repos

### À lire en premier

- `README.md`
- `src/` ou fichier principal
- `app.py`, `main.py`, `App.tsx`, `index.tsx`, `index.js`

### À regarder ensuite

- les noms de dossiers,
- les composants principaux,
- les endpoints,
- les fichiers de config,
- les captures ou assets déjà présents.

## 9. Ce qu'il ne faut pas faire

- lire tout le repo ligne par ligne,
- garder tous les projets,
- écrire des descriptions trop longues,
- mélanger technique et marketing,
- afficher le code brut dans la page projets.

## 10. Résultat attendu aujourd'hui

À la fin de ce planning, tu dois avoir :

1. 3 projets retenus,
2. 3 fiches projet remplies,
3. 3 cartes portfolio prêtes,
4. 1 ordre de priorité,
5. 1 base pour la page projets.

## 11. Conclusion simple

Le vrai problème n'est pas le manque de projets.

Le vrai problème est l'absence de tri et de transformation.

Ce planning sert à :

- sélectionner,
- résumer,
- valoriser,
- afficher.
