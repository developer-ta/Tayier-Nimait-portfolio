## Comment utiliser cet inventaire

1. Partir du nom GitHub pour identifier le vrai projet.
2. Lire le README ou le fichier principal.
3. Decider si le projet va en `Vitrine`, `Secondaire`, `Archive` ou `Prive`.

## Regle

Le nom du dossier local peut etre trompeur.
Le nom du remote Git est la reference prioritaire.
