# Roadmap Source de Verite 2026

_Document central de suivi pour garder une trace stable des décisions, du positionnement et de l'ordre de travail du projet._

## 1. Pourquoi ce document existe

Les discussions de travail peuvent vite devenir volatiles.

Ce document sert de mémoire durable pour:

- conserver les décisions importantes,
- garder la même direction dans le temps,
- éviter de recommencer les réflexions,
- structurer le travail par étapes,
- aligner portfolio, CV et profils publics.

Ce document n'a pas pour but de tout raconter.

Il a pour but de garder la colonne vertébrale du projet.

## 2. Situation actuelle

Le projet est désormais séparé, autonome et orienté vers la valorisation professionnelle du profil.

Les grandes orientations retenues sont:

- un portfolio principal comme pièce maîtresse,
- plusieurs CV ciblés selon l'axe de recherche,
- un profil public cohérent sur LinkedIn et GitHub,
- une documentation stable pour ne pas perdre les décisions.

## 3. Positionnement retenu

La ligne directrice retenue à ce stade est:

> Développeur Python orienté automatisation et orchestration IA, capable de livrer vite des outils métier utiles et fiables.

La signature différenciante peut ensuite venir en second niveau:

- gouvernance IA,
- bio-inspired thinking,
- structuration de workflows,
- culture produit,
- capacité à travailler sur plusieurs domaines.

## 4. Ordre de travail retenu

Le travail doit suivre cet ordre:

1. Portfolio
2. CV
3. Profils publics

### 4.1 Pourquoi le portfolio passe en premier

Le portfolio doit porter:

- la preuve,
- le récit,
- le style,
- la hiérarchie des projets,
- les liens vers les CV et profils publics.

### 4.2 Pourquoi les CV viennent ensuite

Les CV doivent résumer le portfolio et s'adapter à chaque cible.

Ils ne doivent pas inventer une autre histoire.

### 4.3 Pourquoi LinkedIn et les autres profils viennent à la fin

Ils doivent refléter un message déjà validé.

Sinon on corrige dans le mauvais ordre.

## 5. Ce qu'on veut construire

### 5.1 Portfolio

Le portfolio doit montrer:

- qui tu es,
- ce que tu fais,
- ce que tu résous,
- pourquoi c'est crédible,
- comment te contacter.

### 5.2 CV

Le système de CV doit comprendre:

- un CV alternance RNCP 7 IA,
- un CV freelance / contrat,
- éventuellement un CV général tech si nécessaire.

### 5.3 Profils publics

Les profils publics doivent être cohérents avec:

- le positionnement,
- les preuves,
- le ton,
- la cible.

## 6. Règles de conduite

### À faire

- documenter chaque décision importante,
- garder les textes courts et lisibles,
- séparer les axes de recherche,
- travailler par itérations,
- conserver une trace des versions utiles.

### À éviter

- multiplier les messages contradictoires,
- créer trop de versions sans raison,
- mélanger public et privé,
- faire un récit trop large,
- perdre l'ordre des priorités.

## 7. Décisions déjà prises

- Le profil doit être lisible par un recruteur en quelques secondes.
- La base de positionnement doit rester claire et concrète.
- Le portfolio doit précéder les CV.
- Les CV doivent être ciblés.
- LinkedIn et GitHub doivent s'aligner ensuite.
- Les documents sensibles ne doivent pas être exposés publiquement.

## 8. Ce qui doit rester privé

Tout ce qui relève de:

- travail de fond non destiné au public,
- variantes sensibles,
- réflexion détaillée sur le marché et l'âge,
- archives de méthode,
- notes de gouvernance,
- matériaux internes,

doit rester en local ou dans l'espace privé prévu.

## 9. Ce qui peut être publié

Le dépôt public peut montrer:

- le portfolio final,
- les sections utiles à la candidature,
- la documentation publique minimale,
- les éléments qui aident à comprendre le produit,
- les pages d'aide nécessaires à l'utilisateur final.

## 10. Définition du succès

On considérera que le travail avance bien si:

- le portfolio est lisible rapidement,
- les CV sont cohérents avec le portfolio,
- les profils publics racontent la même histoire,
- la documentation permet de reprendre le travail sans perte,
- les éléments privés restent hors du dépôt public.

## 11. Mode de mise à jour

Chaque fois qu'une décision importante est prise, on peut ajouter:

- une nouvelle section,
- une note de version,
- ou un mini journal de décision.

Format recommandé:

- date,
- décision,
- raison,
- impact sur le travail.

## 12. Résumé de travail

La stratégie actuelle est:

- documenter pour garder la mémoire,
- construire le portfolio en premier,
- décliner ensuite les CV,
- puis harmoniser LinkedIn et les autres profils,
- tout en gardant une séparation nette entre public et privé.
